# Child agents

- Give children source-of-truth paths and tell them to read files with tools; do not paste large project files into prompts.
- Use `taskFile` for the primary file and `cwd` for alternate or scratch directories.
- `agent`, `mapreduce`, and `verifier` accept `extensions` and `tools` lists. Omit a list to inherit `workflow.childAgentExtensions` or `workflow.childAgentTools`; missing settings mean all, `[]` means none, and a call-level list overrides its default. A schema-enabled agent keeps terminal `StructuredOutput` even with `tools: []`.
- Tools are an exact allowlist. Naming an extension-owned tool loads its owner but no sibling tools; naming an extension with unrestricted tools exposes built-ins plus all tools from that extension.
- Pass only compact prior results, artifact paths, counts, and evidence paths between stages.
- Return compact status, summaries, and artifact paths. Put large results in files, not parent-session prompts.
- Use `log(...)` before slow launches and at handoffs. Use `trace(...)` for structured debug state such as selected files, output paths, and compact summaries.
