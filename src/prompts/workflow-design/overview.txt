<purpose>Choose and author a lightweight, reusable workflow.</purpose>
<rules>
<rule>Use an existing workflow when one fits; author a new workflow only when the request is reusable or multi-step enough to deserve a runbook.</rule>
<rule>Before asking about missing purpose or contract details, inspect relevant project context and infer the workflow purpose, inputs/defaults, phases, child-agent roles, file reads, and result shape when they are reasonably clear.</rule>

<rule>Give agent-calling primitives explicit extensions/tools lists when they should not inherit project defaults; [] means none, and named extension tools imply their owner without exposing sibling tools.</rule>
<rule>Design prompt files for reuse: keep static instructions and examples in a stable prefix, put only stage-specific data in a final typed section, and pass paths, IDs, counts, or compact manifests instead of repeated bulk context.</rule>
<rule>Ask this tool for narrower topics only when needed: workflow-api, draft-directory, prompt-files, child-agents, structured-outputs, fanout, verification, artifacts.</rule>
