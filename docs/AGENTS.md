## Purpose

`docs/` holds durable specs and product/architecture notes for the workflow extension.

## Rules

- Keep docs crisp, current, and consistent with code and README examples.
- Update specs when workflow primitives, storage layout, commands, review behavior, or sandbox rules change.
- Do not bury agent-only coding rules here; put those in `../agent_docs/INDEX.md`.
- Link or name the source files that implement documented behavior.
