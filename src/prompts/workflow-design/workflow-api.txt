# Workflow API

- `workflow.js` exports static `metadata` and a default async workflow function. Metadata needs `name`, `description`, `inputInstructions`, and `phases`.
- Document the default function with JSDoc covering purpose, input fields/defaults, phases, child agents, file reads, and result shape.
- Supported workflow primitives are rendered below from the runtime registry with short descriptions and signatures; request narrower guidance topics for examples only when needed.
- `agent`, `mapreduce`, and `verifier` accept `extensions?: string[]` and `tools?: string[]`. Omit to inherit settings, use `[]` for none, and use an exact tool list when only named tools should be exposed. Schema-enabled agents always receive terminal `StructuredOutput`.
- `propose_workflow` validates inline and top-level `const` capability lists plus inherited defaults against Pi metadata and saves nothing on error.
- Receive workflow input through `workflow({ field, optional = default })` parameters.
- Workflows cannot import modules or use ambient Node globals. Use file helpers and workflow-owned templates for files and artifacts.

```js
export const metadata = {
  name: "review",
  description: "Review selected files",
  inputInstructions: "Resolve files from explicit path mentions; use remaining prose as focus.",
  phases: [{ title: "review" }, { title: "synthesis" }],
};
```
